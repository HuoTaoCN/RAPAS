# GovInsight-AI (RA) 重复诉求智能分析系统 / Smart Repetitive Appeal Pattern Analysis System

`GovInsight-AI RA` 是专门面向政务热线（如 12345）管理场景设计的**重复诉求智能分析引擎**。它专注于解决政务热线中“同人高频重复诉求”的甄别难、定性难、处置难等痛点。

---

## 🧠 核心研判计分机制 (Scoring Mechanism)

系统引入 LLM 语义理解能力，对每一组重复诉求进行多维度“计分”与定性研判：

### 1. 行为模式分类 (A-H 八大维度)
系统将诉求人的重复行为精准归类，作为处置的主要依据：
*   **【A类】真实高频诉求**：问题持续未解，正当反复。**（重点保障，计分权重：高）**
*   **【B类】情绪驱动型**：不满情绪导致短时间集中提交。
*   **【C类】衍生扩展型**：因核心诉求不满，延伸投诉关联问题。
*   **【D类】代理集中型**：代表群体统一反映共同问题。
*   **【E类】功能测试型**：有意设计场景验证系统响应。
*   **【F类】职业诉求型**：以施压、曝光或获取补偿为目的。
*   **【G类】恶意骚扰型**：以占用资源、干扰考核为目的。**（风险预警，计分权重：高）**
*   **【H类】特殊关怀型**：疑似存在精神健康或认知问题。**（人文关怀，审慎判定）**

### 2. 多维度计分因子
*   **时间规律分**：分析提交间隔（24h内/48h外）、频率曲线（爆发式/匀速式），判断是否符合自然诉求规律。
*   **内容相似度**：通过语义向量比对，识别“模板化复制”与“带有新事实的递进描述”。
*   **关联广度分**：识别诉求是否跨部门、跨领域扩散，评估其“衍生性”。
*   **结论置信度 (Confidence Score)**：AI 结合证据链完整性，输出 0-100 的置信度得分，辅助人工快速定位。

---

## ✨ 核心价值与功能

### 📊 深度分析能力
*   **基础统计**：自动提取总量、周期、涉及部门及高相似工单分布。
*   **时间链路视图**：把重复诉求从“列表”转成“事件链”，识别递进关系。
*   **智能报告生成**：自动输出包含执行摘要、统计分析、行为研判在内的专业 Markdown 报告。

### 🛡️ 分级处置建议
系统根据研判得分，输出三层处置建议：
1.  **针对诉求人**：情绪安抚、专人督办或法律告知。
2.  **针对热线系统**：预警规则设置、数据隔离或满意度标注。
3.  **针对办理部门**：跨部门联办、流程漏洞修复或服务质量改进。

## 🏗️ 系统架构与技术栈

*   **架构**: 全栈 Serverless (Cloudflare Pages + Functions)
*   **前端**: Native HTML5/CSS3/JS, Marked.js
*   **后端**: Hono Framework, OpenAI API (Qwen/GPT)
*   **模型**: 默认使用 Qwen-Max/Plus 级别模型，支持 OpenAI 兼容接口

---

## ⚠️ 审慎声明

本系统分析结论（包括 A-H 分类及置信度得分）仅作为政务管理辅助参考。对于涉及诉求人权益的最终处置决定，**必须经由人工审核确认**。系统特设“H类”标注，旨在引导管理人员关注诉求人可能的特殊困难，体现政务服务的人文关怀。
